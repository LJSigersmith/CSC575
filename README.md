# CSC575 : Open Source Software Development

Welcome to **CSC575: Open Source Software Development**. This course introduces students to the principles, practices, and culture of Free and Open Source Software (FOSS). Through readings, discussions, and hands-on project work, students will learn how open source communities collaborate, build software, and sustain projects over time.

Students will explore the history and philosophy of FOSS, including licensing and copyright considerations, version control, and common workflows used in open source projects. The course also emphasizes **Humanitarian Free and Open Source Software (HFOSS)**, helping students understand how open source can be used to address real-world and social-impact challenges.

A major component of the course is a **team-based open source project**. Students will evaluate existing open source projects, select a project that aligns with their interests and skills, and contribute through activities such as documentation, testing, design, or code contributions. Along the way, students will present their progress and reflect on their learning.

This GitHub repository serves as the **central hub for course materials**, including the syllabus, assignments, project deliverables, and readings. Students will use GitHub throughout the semester to access materials and submit their work, gaining practical experience with tools commonly used in open source development.

# Candidate Projects

- [Glucosio](https://gitlab.com/glucosio) – Open source Android app for diabetes tracking. Appears moderately active with Android focus; suitable for mobile-focused contributions.
- [Internet Archive BookReader](https://github.com/internetarchive/bookreader) – Web-based reader for scanned books. Large ecosystem with strong contributor base; potentially complex but active.
- [Wagtail](https://github.com/wagtail/wagtail) – Django-based CMS used for content management. Mature and active Python project with good documentation.
- [The Odin Project](https://github.com/TheOdinProject/theodinproject) – Open source web development curriculum platform. Good for frontend contributions.
- [TEAMMATES](https://github.com/TEAMMATES/teammates) – Peer feedback management system for education. Strong academic focus and structured contribution process.
- [Zulip](https://github.com/zulip/zulip) – Threaded team chat platform. Very active large-scale project with strong contributor onboarding.
- [wger](https://github.com/wger-project/wger) – Open source fitness tracker. Moderate scale, good documentation, labeled beginner issues.
- [Open Library](https://github.com/internetarchive/openlibrary) – Public digital library platform. Active Python-based project aligned with HFOSS.
- [CourtListener](https://github.com/freelawproject/courtlistener) – Public access to legal opinions and filings. Strong HFOSS alignment and highly active.
- [Uwazi](https://github.com/huridocs/uwazi) – Human rights document management platform. Strong social impact focus and active TypeScript codebase.
- [Signal](https://github.com/signalapp/Signal-Android) – End-to-end encrypted messaging platform focused on privacy and secure communication. Highly active Android codebase with strong contribution guidelines; technically demanding but well-documented and impactful.
