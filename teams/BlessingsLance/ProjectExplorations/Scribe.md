## 12. Scribe-Android

### Repository
> [Scribe-Android Repo](https://github.com/scribe-org/Scribe-Android)

### Homepage
> [Scribe Homepage](https://scri.be/)

### Product
> A pack of Android keyboards for language learners. Features include word translation (beta), verb conjugation, and word annotation: giving users contextual grammar assistance directly while typing. All feature data is sourced from Wikidata and stored on-device, so Scribe is fully offline-capable and collects zero usage data. Also available on iOS, with Desktop planned.

### Target User Community
> Language learners, multilingual users, and anyone who wants grammar assistance while typing on Android.

### Activity
> ~460 commits, last commit December 2025, ~20 open issues (17 labeled as needing help).

### Getting Started
> The [CONTRIBUTING.md](https://github.com/scribe-org/Scribe-Android/blob/main/CONTRIBUTING.md) is the primary guide. It covers forking and cloning the repo, setting up Android Studio, running the app on an emulator, pre-commit hooks for code quality, writing unit tests, and the PR process using GitHub Flow and Conventional Commits. The README also links to an [architecture diagram](https://github.com/scribe-org) for understanding how Scribe-Android fits into the broader Scribe organization. Setup is straightforward: clone, open in Android Studio, run on a Pixel emulator. Very clear and beginner-oriented.

### Welcoming to Newcomers
> Exceptionally welcoming. The contributing guide explicitly states: *"Those new to Kotlin or wanting to work on their Kotlin skills are more than welcome to contribute! The team would be happy to help you on your development journey."* The project uses a public [Matrix chat](https://matrix.to/#/#scribe_community:matrix.org) for real-time communication, holds bi-weekly developer syncs, and actively maintains `good first issue` labels — with 10+ currently open. After a few merged PRs, contributors are invited to discuss becoming an organization member or maintainer.
